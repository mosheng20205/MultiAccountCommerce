# 独立的浏览器宿主面板方案

## 目标

在当前 `emoji_window_cpp + 易语言` 的电商多账号浏览器草图结构上，采用：

- 业务状态独立：每个环境独立一个浏览器实例、缓存目录、代理、指纹、Cookie 状态
- 界面框架共享：右侧仍然只有一套固定布局（信息栏、工具栏、浏览器外框）
- 显示宿主独立：在浏览器内容区内部，为每个环境分配一个独立的浏览器宿主面板

这样做的结果是：

- 不会串账号状态
- 切换环境时不需要销毁重建整个右侧界面
- 可以保留环境页面状态，切回来直接继续使用
- 结构和当前草图代码最接近，改造成本最低

---

## 一、整体结构

当前右侧布局继续保留：

- `右侧面板`
- `信息栏面板`
- `工具栏面板`
- `浏览器面板`
- `浏览器外框`

在 `浏览器外框` 内新增：

- `浏览器宿主区面板`

`浏览器宿主区面板` 下再按环境动态创建多个子面板：

- `环境宿主面板_A`
- `环境宿主面板_B`
- `环境宿主面板_C`

每个 `环境宿主面板` 只绑定一个环境，对应一个独立浏览器实例。

切换环境时：

- 隐藏旧环境宿主面板
- 显示新环境宿主面板
- 刷新顶部信息栏和工具栏状态

不建议在切换时把同一个浏览器实例来回搬运到不同环境下，也不建议让多个环境共用一个浏览器实例。

---

## 二、环境表结构

不要再只靠树节点 ID 管理环境。应增加一张环境表，把环境配置和运行时状态统一放进去。

建议使用 `环境记录` 数据类型：

```text
.数据类型 环境记录
    环境ID, 整数型
    节点ID, 整数型
    分组名, 文本型
    环境名, 文本型
    域名, 文本型
    代理名, 文本型
    状态文本, 文本型
    评分, 整数型

    启动网址, 文本型
    缓存目录, 文本型
    指纹配置ID, 文本型
    浏览器标识, 文本型
    Cookie路径, 文本型

    浏览器宿主面板, 整数型
    浏览器句柄, 整数型
    浏览器状态, 整数型
    是否后台保活, 逻辑型
    是否可见, 逻辑型

    最后地址, 文本型
    最后标题, 文本型
```

### 字段说明

- `环境ID`：稳定主键，只增不复用
- `节点ID`：当前树形框里对应的节点句柄，树重建后会变化
- `浏览器标识`：建议固定为 `env_环境ID`，用于和浏览器事件回调绑定
- `浏览器宿主面板`：该环境专属的宿主 Panel 句柄
- `浏览器句柄`：实际浏览器实例句柄或对象句柄
- `浏览器状态`：建议统一枚举值

建议浏览器状态枚举：

```text
0 = 未创建
1 = 创建中
2 = 运行中
3 = 隐藏保活
4 = 关闭中
5 = 已关闭
6 = 异常
```

### 程序集变量建议新增

```text
.程序集变量 环境表, 环境记录, , "0"
.程序集变量 当前环境ID, 整数型
.程序集变量 当前显示环境ID, 整数型
.程序集变量 下一个环境ID, 整数型
.程序集变量 浏览器宿主区面板, 整数型
.程序集变量 最大保活浏览器数, 整数型
```

建议初始化：

```text
下一个环境ID ＝ 1001
当前环境ID ＝ 0
当前显示环境ID ＝ 0
最大保活浏览器数 ＝ 3
```

## 三、节点 ID -> 浏览器实例映射

不要直接做：

- `节点ID -> 浏览器句柄`

正确做法是两层映射：

1. `节点ID -> 环境ID`
2. `环境ID -> 环境表 -> 浏览器句柄 / 宿主面板`

原因：

- 树形框重建后，节点 ID 可能变化
- 浏览器实例生命周期比树节点更长
- 环境切换、主题切换、树重建都不能影响浏览器实例归属

### 推荐辅助子程序

```text
.子程序 取环境索引_按环境ID, 整数型
.参数 环境ID, 整数型

.子程序 取环境索引_按节点ID, 整数型
.参数 节点ID, 整数型

.子程序 取环境ID_按节点ID, 整数型
.参数 节点ID, 整数型

.子程序 更新环境节点ID
.参数 环境ID, 整数型
.参数 新节点ID, 整数型
```

### 建树时的正确顺序

1. 先创建环境记录，生成稳定 `环境ID`
2. 再创建树节点，拿到 `节点ID`
3. 把 `节点ID` 回填进环境记录

也就是说：

- 环境表是数据源
- 树形框只是环境表的一个界面映射

## 四、浏览器宿主区设计

### 新增面板层级

在当前 `浏览器外框` 内，建议新增这一层：

```text
浏览器外框
  ├ 标签_浏览器标题
  ├ 标签_浏览器说明
  ├ 标签_浏览器占位
  └ 浏览器宿主区面板
       ├ 环境宿主面板(环境ID=1001)
       ├ 环境宿主面板(环境ID=1002)
       └ 环境宿主面板(环境ID=1003)
```

### 作用划分

- `浏览器外框`：公共边框和标题区
- `浏览器宿主区面板`：浏览器内容区总容器
- `环境宿主面板`：每个环境的浏览器专属挂载点

### 为什么这样最稳

- 右侧整体布局只维护一套
- 每个环境的浏览器都在自己的面板里，不互相干扰
- 切换时只做 `ShowWindow` / `隐藏窗口`
- 后续接 FBro/CEF 时可以直接把 `parent_window` 指到该环境宿主面板

## 五、切换环境时的显示 / 隐藏流程

建议新增一个总入口：

```text
.子程序 切换到环境
.参数 目标环境ID, 整数型
```

### 标准流程

1. 根据 `目标环境ID` 找到环境记录
2. 如果当前显示环境就是它，只刷新信息，不重复切换
3. 如果当前显示环境不为 0，先隐藏旧环境宿主面板
4. 如果目标环境宿主面板未创建，先创建宿主面板
5. 如果目标环境浏览器未创建，按需启动浏览器
6. 显示目标环境宿主面板
7. 刷新顶部信息栏、窗口标题、工具栏可见状态
8. 更新：
   - `当前环境ID`
   - `当前环境节点ID`
   - `当前显示环境ID`

### 伪代码

```text
.子程序 切换到环境
.参数 目标环境ID, 整数型
.局部变量 目标索引, 整数型
.局部变量 旧索引, 整数型

目标索引 ＝ 取环境索引_按环境ID (目标环境ID)
.如果真 (目标索引 ＝ -1)
    返回 ()
.如果真结束

.如果 (当前显示环境ID ＝ 目标环境ID)
    刷新环境页_按环境ID (目标环境ID)
    返回 ()
.如果结束

.如果 (当前显示环境ID ≠ 0)
    旧索引 ＝ 取环境索引_按环境ID (当前显示环境ID)
    .如果真 (旧索引 ≠ -1)
        隐藏环境浏览器 (环境表 [旧索引].环境ID)
    .如果真结束
.如果结束

确保环境宿主面板 (目标环境ID)
确保环境浏览器 (目标环境ID)
显示环境浏览器 (目标环境ID)
刷新环境页_按环境ID (目标环境ID)
当前显示环境ID ＝ 目标环境ID
当前环境ID ＝ 目标环境ID
```

### 切到模块页时

如果点击的是：

- `代理管理`
- `分组管理`
- `RPA自动化`
- `系统设置`

则流程是：

1. 隐藏当前环境宿主面板
2. 不关闭浏览器
3. 显示模块占位页
4. `当前显示环境ID ＝ 0`
5. `工具栏显示 ＝ 假`

这意味着：

- 浏览器还活着，只是不显示
- 下次切回环境可以继续使用原状态

## 六、启动浏览器时的生命周期管理

建议把“启动浏览器”按钮逻辑改成环境级别的 `确保环境浏览器`。

```text
.子程序 确保环境浏览器
.参数 环境ID, 整数型
```

### 启动规则

如果环境浏览器状态是：

- `未创建`
- `已关闭`
- `异常`

则执行创建流程。

如果环境浏览器状态是：

- `运行中`
- `隐藏保活`

则直接显示，不重复创建。

### 创建流程建议

1. 确保宿主面板已存在
2. 生成浏览器标识 `env_环境ID`
3. 读取环境配置：
   - 缓存目录
   - 启动网址
   - 代理
   - 指纹配置
4. 创建浏览器实例，父窗口挂到 `环境宿主面板`
5. 创建成功后回写：
   - `浏览器句柄`
   - `浏览器状态 ＝ 运行中`
   - `是否可见 ＝ 真`
6. 创建失败则：
   - `浏览器状态 ＝ 异常`
   - `状态文本 ＝ 启动失败`

### 与 FBro/CEF 的对应关系

如果你后面用 FBro 这类浏览器库，建议：

- `user_flag = 浏览器标识`
- `parent_window = 环境宿主面板`
- `cache_path = 缓存目录`

这样浏览器事件里就能通过 `user_flag` 找回对应环境。

## 七、关闭环境浏览器的生命周期管理

建议新增：

```text
.子程序 关闭环境浏览器
.参数 环境ID, 整数型
```

### 关闭规则

关闭的是“当前环境的浏览器实例”，不是删除环境本身。

### 关闭后建议回写

- `浏览器句柄 ＝ 0`
- `浏览器状态 ＝ 已关闭`
- `是否可见 ＝ 假`
- `状态文本 ＝ 待启动`
- `最后地址 / 最后标题` 可以保留

### 当前正显示时的处理

如果关闭的是当前显示环境：

1. 隐藏该环境宿主面板
2. 显示占位标签
3. `当前显示环境ID ＝ 0`
4. `标签_浏览器说明` 改为“当前环境浏览器已关闭”

### 不建议的做法

- 不建议关闭环境时顺便删除环境记录
- 不建议关闭一个环境时影响其他环境浏览器
- 不建议把“关闭”实现成“只是隐藏”但状态仍显示运行中

---

## 八、删除环境时的生命周期管理

建议新增：

```text
.子程序 删除环境
.参数 环境ID, 整数型
```

### 删除前必须做的事

1. 如果该环境浏览器还在运行，先关闭浏览器
2. 销毁该环境宿主面板
3. 删除树节点
4. 从环境表中删除该记录

### 删除后的回退策略

按这个优先级选择下一个目标：

1. 同分组下一个环境
2. 同分组上一个环境
3. 其他任意一个环境
4. 没有环境则回到“环境切换”占位页

### 删除时必须同步清理

- `当前环境ID`
- `当前环境节点ID`
- `当前显示环境ID`
- 宿主面板句柄
- 浏览器句柄

### 删除环境时不应发生的问题

- 树节点删了，但环境表还在
- 环境表删了，但浏览器还活着
- 当前环境已删，但顶部信息栏还是旧环境数据

---

## 九、后台保活策略

你当前界面里已经有“打开后台”按钮，这个正好可以作为保活开关。

### 推荐策略

- 切换环境时，默认不关闭旧环境浏览器
- 旧环境浏览器改为 `隐藏保活`
- 切回来时直接恢复显示

### 对应状态

- `运行中`：当前可见并可操作
- `隐藏保活`：不可见，但实例还在
- `已关闭`：实例已销毁

### 为什么建议保活

多账号场景下，保活可以保留：

- 登录态
- 页面滚动位置
- 表单输入状态
- 中间业务页面上下文

### 内存控制建议

避免无限保活，建议加一个上限：

```text
最大保活浏览器数 ＝ 3 或 5
```

超过上限后：

- 关闭最久未激活的隐藏环境
- 保留当前环境和最近使用的几个环境

后续可以再补：

- 最后激活时间
- 最近使用顺序
- 自动淘汰隐藏实例

## 十、对当前易语言代码的改造建议

### 1. 先保留当前主布局不动

继续保留：

- `左侧面板`
- `右侧面板`
- `信息栏面板`
- `工具栏面板`
- `浏览器面板`
- `浏览器外框`

不要一开始就推翻当前草图布局。

### 2. 在 `创建界面` 中增加一个新面板

新增：

```text
浏览器宿主区面板 ＝ 创建面板 (浏览器外框, 0, 0, 100, 100, 颜色_白)
```

并在 `重新布局` 中给它单独定位。

### 3. 树回调不要直接写死页面内容

现在你的 `树形框选中回调` 是：

- 判断节点 ID
- 直接调用 `刷新环境页`

建议改成：

- 判断节点是不是环境节点
- 如果是环境节点：先取 `环境ID`，再调用 `切换到环境`
- 如果是模块节点：调用 `切换到模块页`

### 4. 按钮逻辑要改成面向环境表

现在的“启动浏览器”只是改标签文本，后面应改成：

- 取 `当前环境ID`
- 调用 `确保环境浏览器 (当前环境ID)`
- 调用 `显示环境浏览器 (当前环境ID)`

“删除环境”应改成：

- 取 `当前环境ID`
- 调用 `删除环境 (当前环境ID)`

### 5. 主题切换不要重建浏览器实例

你现在有 `应用主题 -> 重建标签控件 -> 重建树形框控件`。

后面即使切换主题，也不要销毁环境浏览器：

- 主题只改公共 UI 颜色
- 浏览器实例继续存在
- 最多只更新宿主区背景色

## 十一、推荐新增的子程序清单

建议按下面这组子程序分层：

```text
初始化环境表
添加环境记录
更新环境节点ID
取环境索引_按环境ID
取环境索引_按节点ID
取环境ID_按节点ID
刷新环境页_按环境ID
切换到环境
切换到模块页
确保环境宿主面板
确保环境浏览器
显示环境浏览器
隐藏环境浏览器
关闭环境浏览器
删除环境
同步环境状态到树节点颜色
清理超限保活浏览器
```

### 最核心的 4 个入口

如果你先做第一版，实现这 4 个就够了：

- `切换到环境`
- `确保环境宿主面板`
- `确保环境浏览器`
- `关闭环境浏览器`

## 十二、最终结论

这套方案的核心是：

- 数据层：每个环境独立
- 浏览器实例层：每个环境独立
- 浏览器宿主面板层：每个环境独立
- 右侧整体界面框架：共享

最适合当前项目的实现方式是：

- 用 `环境ID` 做稳定主键
- 用 `节点ID` 做临时映射
- 每个环境一个独立宿主面板
- 每个环境一个独立浏览器实例
- 切换时只做显示 / 隐藏，不做跨环境复用实例

一句话概括：

**环境独立，布局共享，宿主独立，切换显隐。**
